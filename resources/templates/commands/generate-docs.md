# /generate-docs

Generate documentation for the specified code:

1. **Overview**: What does this code do?
2. **API Reference**: Functions, classes, interfaces
3. **Usage Examples**: How to use the code
4. **Configuration**: Any setup required

Documentation Formats:
- README.md for projects
- JSDoc/TSDoc for code
- API documentation for endpoints
- Architecture docs for systems

Include:
- Clear descriptions
- Parameter documentation
- Return value documentation
- Code examples
- Error handling notes

Output in Markdown format with proper formatting and code blocks.
