# /refactor

Refactor the code to improve:

1. **Readability**: Clear naming, logical structure
2. **Maintainability**: Reduced complexity, better organization
3. **Reusability**: Extract common patterns
4. **Performance**: Optimize where beneficial

Refactoring Techniques:
- Extract Method/Function
- Extract Variable
- Rename for clarity
- Remove duplication
- Simplify conditionals
- Apply design patterns

Rules:
- Preserve existing behavior
- Make incremental changes
- Explain each refactoring step
- Highlight any risks

Provide:
- Refactored code
- Explanation of changes
- Before/after comparison for significant changes
